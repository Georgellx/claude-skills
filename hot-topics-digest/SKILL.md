# 实时热点速览（Hot Topics Digest）

用户触发时，拉取**微博热搜 + 抖音热榜各前 5 条**，并为每一条补全**正文内容**（不只是标题），汇成一份干净、带来源的中文速览。

为什么要补正文：热榜标题往往是黑话或半句话（"土耳其球员急哭了""等你年纪大了就知道原因了"），光看标题根本不知道发生了什么。用户要的是**事件本身的内容**，所以每条都必须落到"到底怎么回事"。

## 两条铁律

1. **不做关键词过滤** —— 榜单前 5 是什么就是什么，不挑、不删、不按主题筛。用户要的是真实榜单原貌。
2. **正文必须有据，不许编** —— 每条的内容摘要都来自联网搜索到的真实信息。搜不到可靠信息的，老实标注"暂未搜到详情"，绝不脑补编造。

## 工作流

### 第 1 步：抓榜单（跑脚本）

运行本 skill 自带的脚本，拿到两平台各前 5 条（免登录、稳定）：

```bash
node "<skill目录>/scripts/fetch-hotlist.mjs" --top 5
```

脚本输出 JSON：`{ stamp, weibo:[...], douyin:[...], errors:[] }`，每条含 `rank / title / hot / tag / search_url`。
- 若某平台 `errors` 里报错（接口偶尔变动），照常处理另一个平台，并在结尾说明哪个没抓到。
- 默认各 5 条；用户若说"多给几个/前 10"，把 `--top` 调大。

### 第 2 步：逐条补正文（联网搜索）

对这 10 条（5+5）里的**每一条**，用 `WebSearch` 搜出"这事到底怎么回事"。可以并发多条一起搜以提速。

- 搜索词用热点标题本身；若标题太短/有歧义，补上平台或"热搜""最新"等词消歧（例：`土耳其 急哭 世界杯`、`大渡河金川水电站 投产`）。
- **务必确认是当前最新的事件**（对照脚本输出的 `stamp` 日期），别拿到同名旧闻。
- 搜索摘要不足以说清时，对最关键/最不清楚的几条用 `WebFetch` 抓一篇权威来源补细节；不必每条都 fetch，控制开销。
- 每条提炼 2–4 句：**发生了什么 + 关键信息（人物/数字/结果） + 为什么火**。

### 第 3 步：输出速览

用下面的固定结构，在对话里直接呈现（默认不写文件）：

```markdown
# 实时热点速览 · <stamp>

## 微博热搜 TOP5
1. **<标题>** `<tag>` · 热度 <hot>
   <2–4 句正文：发生了什么 / 关键信息 / 为什么火>
   来源：<链接>
2. ……（共 5 条）

## 抖音热榜 TOP5
1. **<标题>** `<tag>` · 热度 <hot>
   <2–4 句正文>
   来源：<链接>
2. ……（共 5 条）
```

- 热度用脚本已格式化的口径（万/亿）即可。
- 每条尽量给一个可点的来源链接（搜到的新闻/官方；实在没有就给脚本里的 `search_url`）。
- 末尾一句话点出**今天的整体氛围**（比如"今天体育赛事 + 端午民俗占主导"），帮用户快速抓重点。

## 收尾（可选，别啰嗦）

呈现完速览后，可以**一句话**轻轻递一个后续，让用户选，而不是默认执行：
- 「要不要把这份速览存进知识库？」（存的话放 `D:\KnowledgeBase\Note\热点\` 下，文件名带日期时间）
- 「想针对某条深挖，或筛出跟你 AI/科技/产品方向相关的，我可以再来一轮。」

不要主动加关键词过滤、不要自动存档、不要展开成需求访谈——这个 skill 的价值是**快、准、原貌**。

## 实现备注（给维护者）

- 微博正文、抖音视频正文都需要登录态/签名才能直接爬，**不稳且易坏**。所以脚本只负责抓"免登录就能拿到的榜单"，正文一律走触发时的联网搜索补全——这样平台改了反爬也不影响 skill 主流程。
- 脚本依赖 Node（用户机器有 Node v24）。微博接口需先访问首页拿访客 cookie，脚本已内置处理。
