# 求职意愿确认清单（preference-checklist）

第 2 步的详细追问清单 + 把模糊回答问清楚的技巧。核心：这些是用户的人生选择，必须问清、不替他决定（铁律 1）。

## 必问项（按组问，别一次砸完）

### 1. 地点与通勤
- 目标城市？
- 住在哪 / 能接受的通勤范围（如"≤90 分钟"或"只投某几个区"）？
- 接受远程吗？接受出差 / 外派吗？

### 2. 薪资
- 期望区间？
- **下限是多少**（低于这个数直接不投）？——这是硬标尺，一定要问出一个明确数字。

### 3. 行业
- 想投哪些行业？
- 有没有明确**排除**的行业？

### 4. 岗位与级别
- 目标岗位名（可多个）？
- 能接受的级别范围（只投负责人/总监，还是专家/高级也投）？

### 5. 硬性排除项
常见的，逐一确认：
- 排除外包 / 驻场 / 人力派遣？
- 排除单休 / 大小周？
- 排除明确卡年龄（如"35 岁以下"）或学历的岗位？
- 其他雷区？

### 6. 软偏好（有则问，无则跳过）
- 公司规模 / 性质（国企 / 外企 / 民营 / 创业）
- 稳定性 vs 成长性更看重哪个
- 其他在意的点（加班文化、团队规模等）

## 把模糊回答问清楚的技巧

- **追到可执行为止。** 用户说"薪资高一点"——追问"高一点的下限是多少？低于多少你就不看了？"。模糊偏好没法当筛选标尺。
- **给选项降低回答成本。** 与其问"你想投什么行业"，不如"你更想留在现在的财税/企服领域，还是想往纯 AI 公司转？还是都看看？"
- **区分"想要"和"能接受"。** 理想区间和底线区间不一样，匹配时底线区间才是硬门槛。
- **别诱导。** 不要把自己的判断包装成问题塞给用户（"你应该不想去创业公司吧？"）。中立地问。

## 输出：投递偏好卡

把回答整理成一张卡喂回用户确认，例如：

```
投递偏好卡（请确认）
- 城市/通勤：广州，通勤≤90分钟（住南沙黄阁），可远程
- 薪资下限：15K，低于不投
- 行业：不限
- 岗位/级别：AI产品经理为主，负责人/高级均可
- 硬性排除：外包/驻场、单休、明确要求35岁以下
- 节奏：批量，每次10来个
```

确认无误后，这张卡就是第 3 步每个职位的硬标尺。
