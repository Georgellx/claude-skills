# 人物画像指南（profiling-guide）

第 1 步"读简历→人物画像"的详细维度和技巧。目标：从简历里读出一个**立体的、有依据的候选人画像**，作为后续岗位匹配的地基。

## 提炼原则

- **每个结论都要有简历里的证据。** 不写"沟通能力强"这种谁都能套的话，写"主导跨 5 省 26 家子公司的协同系统落地，建立跨省协同机制"——让优势可被验证。
- **从动词和数字里挖。** "主导""从 0-1 搭建""推动…提升 X%"这类表述，是判断他真正做过什么、处在什么层级的信号。
- **看轨迹，不只看最近一份。** 一个人的职业故事藏在岗位的演变里：是越做越深（专家），还是越管越宽（管理），还是换了赛道（转型）。

## 画像应包含的维度

1. **一句话定位**：市场视角的候选人画像。例："16 年经验的企业级 B 端产品负责人，财税/低代码/AI 三栖"。
2. **核心优势（3-5 条，带证据）**：每条 = 能力点 + 简历证据。
3. **经验主线**：用一两句讲清他的职业故事和当前所处阶段。
4. **能力标签**：10 个左右便于岗位匹配的关键词。区分：
   - 领域标签（财税、电网、OA、低代码）
   - 能力标签（B端产品、需求分析、数据治理、项目管理）
   - 技术/AI 标签（RAG、知识图谱、LSTM、Agent）
5. **适合投的岗位类型（核心产出）**：推断 3-5 类适合的岗位方向，每类一句理由。这是用户最想要的答案。可分梯队：
   - **最匹配**（经验直接对口、命中率高）
   - **可拓展**（凭可迁移能力够得着、但需要在简历/沟通里补论证）
   - **够不着 / 不建议**（诚实标出，避免浪费）
6. **短板与天花板（诚实）**：
   - 可能被硬卡的点（年龄、学历、某类硬性证书/经验缺失）
   - 简历当前的弱项（如某段经历表述模糊、缺量化结果）
   - 哪些方向是天花板，投了大概率被拒

## 画像之后

画像不是终点，是用来和用户**校准认知**的。如果用户看完画像觉得"你说的某条优势其实我没那么强""我其实更想往某个方向走"，要据此调整——画像是假设，用户的反馈是修正信号。
