# 内容消化官（Content Digest）

把用户在学习的博主文章 / 链接 / 长文本，**经过价值评估后**消化进本地知识库，并在同一次阅读里把"能复刻变现的打法 / 观察到的真实需求 / 被激发的点子"一起捞出来存成产品想法。

> **为什么是一个 skill 而不是两个**：用户读博主干货的真实目的不是存知识，是找"能复刻赚钱的已验证打法"。文章是手段。一篇文章常常同时是「知识」和「可复刻的点子」——分开处理会让"想法"那一半在读的当下漏掉。

**这个 skill 的灵魂三句话**：
1. **先判断价值再决定怎么存**（知识部分要评判真伪）。
2. **一切产物各归其位、单一来源、不重复**（知识 / 提示词 / 想法 三条流分开落地）。
3. **复刻要锚在真实样本上**（产品想法默认追问"成了没有、有什么证据"，把想法钉到地面，而不是空想）。

---

## 第一步永远是：判模式

- **有外部文章正文 / 链接 / Clippings 路径 / 一大段别人的内容** → 【消化模式】，走下面的完整流程。
- **只有用户自己一句话的想法 / 需求、没有文章** → 【速记模式】，直接跳到「速记模式流程」，低摩擦捕捉，**别盘问**。

拿不准时：问一句"这是要我消化的文章，还是你自己冒出来的想法？"

## 铁律

1. **先判模式**（消化 / 速记），别把低摩擦的闪念捕捉拖进重流程。
2. **消化模式先分诊后入库**：任何文章先给"价值档位（A/B/C）+ 一句理由"，让用户看到判断，不黑箱。
3. **各归其位、单一来源**：文章正文→`文章笔记`；可复制提示词→`提示词目录`（只存一份，文章里放指针双链 `[[01-内容写作]]`）；想法/需求→`产品想法`。绝不两处各抄一份。
4. **想法捕捉低摩擦、不劝退**：捕捉阶段别盘问、别泼冷水。风险写进"风险与难点"栏即可。给多少信息存多少，空缺留"待补"。
5. **复刻锚真实样本**（关键）：产品想法——尤其"博主来源"的——必须带「已验证样本 / 复刻证据」。消化模式下博主常自报粉丝/收入/销量/案例，**直接抽出来填**；没有就如实写"待验证"，不替它编。这一栏是为了守住"复刻死守找已验证样本"那步。
6. **原文是一手资料，默认保留**；重写/替换要保守，宁可多留。**不臆造**：抓不到、字段缺，就如实说、留空、要正文。
7. **保守双链**：只按概念/观点连，0–3 条，低置信就不连。双链一律 `"[[真实文件名]]"`。
8. **分类我来扛**：以"知识体系架构师"视角决定要不要新增分类、归到哪，报告里写理由（方便事后否决），不反复丢回给用户。
9. **入库即清理**：源文件来自 `Clippings` 的，验收通过后删除；其它目录的文件不碰。

### frontmatter 铁律（全库统一，别再写坏写法）
- `type` 只从 **8 值受控词表**里选：`官方快照 / 文章笔记 / 书籍 / 自我认知 / 产品想法 / 复盘 / 索引 / 提示词`。
- `tags` 用 **YAML 块列表**（换行 `  - x`），不要内联 `[a, b]`。
- `related_notes` 用**加引号块列表**、每条一行 `  - "[[真实文件名]]"`，目标必须是真实文件名（短名要对齐，如 `如何找到想做的事` → `如何找到想做的事（八木仁平）`）。**不要**写内联 `[[A], [B]]`（YAML 无效、不渲染、不进 Graph）。
- 空字段**删掉整行**，别留 `original_title: ""` 这种空占位。

## 消化模式流程（有文章时）

### 第 0 步：去重
扫一眼 `文章笔记\` 有没有同主题/同源/同标题。已存在就问"更新还是跳过"，不默默建重复。

### 第 1 步：取材
- 粘贴正文→直接用。链接→抓正文、分离页面噪音（导航/计数/侧栏/评论），**先存干净中间稿**确认提示词块/代码块/长清单没丢。Clippings 路径→读文件。抓不到→明说"读不到"，要正文，**不猜**。

### 第 2 步：价值分诊（核心）
给原文定档（标准见 [REFERENCE.md](references/REFERENCE.md)），写一句"价值判断+理由"：
- **A 高价值** → 完整保留原文 + 「方法解读与验证」层。
- **B 中价值** → 保留原文 + 一版"干净重写/要点重述"。
- **C 低价值** → 不存全文，只存 摘要+核心观点+关键引用+来源链接+我的提炼。

### 第 3 步：拆解三类产物
固定产出 `文章摘要 / 核心观点 / 可复用句子`；有方法论/可验证论断时加「方法解读与验证」（大白话重述 + 真伪/适用性点评，与体量成比例）。然后**主动问这两件事**：
- **有没有可直接复制的提示词？** → 进提示词目录（第 4 步）。
- **有没有"我能复刻变现的打法 / 观察到的真实需求 / 被激发的点子"？** → 进产品想法（走「产品想法子流程」）。一篇可能同时出 0~3 类，别硬凑也别漏。

### 第 4 步：分类路由（架构师视角）
- 文章→`文章笔记\`，靠 tag 归类。提示词→按分类表路由到 `0X-*.md`，**追加不覆盖**，文章里留指针双链。新增分类规则见 [REFERENCE.md](references/REFERENCE.md)。

### 第 5 步：写入与双链
- 按模板写 `文章笔记\<标题>.md`；`关联笔记` 逐条写连接理由；提示词追加到分类文件末尾。

### 第 6 步：验收 + 清理
- 跑「验收清单」（见 [REFERENCE.md](references/REFERENCE.md)）。给简报：建/改了哪些文件（可点链接）、价值档位、分类与新增理由、双链了哪些、**抽出了几个想法/需求**。Clippings 源文件验收后删。

### 产品想法子流程（消化模式抽出想法时，或速记模式直接用）
1. 提取核心，起一个简洁短名（中文优先，如 `换词不换曲生成器`）。
2. 在 `产品想法\<短名>.md` 用产品想法模板（见 [TEMPLATES.md](references/TEMPLATES.md)）建文件。**重点填「已验证样本 / 复刻证据」**：谁的打法、成了没有（真实数据）、这是"想法"还是观察到的"真实需求"——消化模式从文章里抽，速记模式留"待补"。
3. `idea_source` 填 `复刻`（抄博主）或 `原创`（自己冒的）。
4. **更新索引看板**：回 `00-想法总览.md` 在表格加一行 `| [[<短名>]] | <一句话> | <状态emoji+文字> | <最大拦路虎> |`（必做，漏更新等于白存）。

## 速记模式流程（只有用户自己的想法/需求时）

灵感稍纵即逝，**第一原则是快**。别盘问、别评判、别劝退。
1. 听懂想法，定短名。
2. 判断「完整文件」还是「一行速记」：认真描述了→完整文件；只甩一句/在赶时间/说"先记一行"→只在 `00-想法总览.md`「速记区」加 `<!-- 日期 一句话 -->`。拿不准→倾向完整文件。
3. 完整文件走「产品想法子流程」第 2–4 步（复刻证据栏留"待补"，不追问）。
4. 回话：存到哪（可点链接）、什么状态、索引已更新。最多最后**轻轻**问 1 个最关键空缺（"这主要解决谁的什么问题？不答也行，先存着了"），不长篇复述。

## 编码与写入注意（Windows）
- 中文务必 UTF-8，写完抽查没乱码。追加到现有文件**用 Edit 锚定末尾**，不要 Write 整文件覆盖（会丢内容）。路径用反斜杠，但双链里只写笔记名 `[[标题]]` 不带路径。
