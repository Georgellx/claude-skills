## 价值分诊标准

- **A 高价值**：原创观点 / 信息密度高 / 不可替代（一手实践、独家数据、成体系的方法论）。
- **B 中价值**：有干货但啰嗦、AI 味重、软文夹带、结构松散——值得留，但需要给一版干净重述。
- **C 低价值**：水文 / 纯营销 / 蹭热点 / 一段话能概括 / 全是已知常识。不灌全文，只留提炼 + 链接。

判断时看：有没有"只有这篇才有"的东西？去掉包装后还剩多少干货？读者拿走后能不能复用？

> 复刻视角额外加一问：**这篇里有没有"已经被验证赚到钱/做出量"的打法？** 有就一定要抽进产品想法的「复刻证据」栏——这才是用户读文章的真正目标。

## 新增分类决策（架构师视角，用户已授权我来定）

默认**不新建**——优先塞进现有分类，分类膨胀是知识库头号杀手。

只有同时满足才新建：
1. 内容确实塞不进任何现有分类（不是"沾边"，是"错位"）；
2. 大概率会**重复出现**（不是孤例）；
3. 新分类边界清晰、和现有分类不重叠。

新建对象：
- 新的**提示词用途** → 在 `提示词目录\` 建 `08-xxx.md`（沿用现有结构），并在 `00-总目录.md` 表格 + 快速入口登记。建好的分类页 frontmatter 用 `type: 提示词`。
- 新的**知识领域**（文章层面）→ 一般只加 tag；只有出现一整个新领域且会持续积累，才在 `知识资产\` 下开新顶层板块。

新建后必须在简报里写明"新增了 X 分类，因为……"，让用户能一眼否决。

## 验收清单

1. 模式判断正确（消化 / 速记）。
2. 消化模式：价值档位已给出且有理由。
3. 文章笔记符合模板，frontmatter 按新规范（type 8 值 / tags 块列表 / related_notes 加引号真实文件名 / 无空占位）。
4. A/B 档原文已保留；C 档至少有来源链接 + 关键引用。
5. 提示词路由正确、单一来源、文章里是指针不是副本。
6. **想法/需求已识别并落地**：该抽的打法/需求进了产品想法，「复刻证据」栏已填或如实标"待验证"；`idea_source` 已标复刻/原创；`00-想法总览` 索引已更新。
7. tag 3-5 个、利于检索；双链概念可信、不过连。
8. 新增分类（若有）已更新 `00-总目录` 并写明理由。
9. 中文无乱码；网页导入先存中间稿，提示词/代码/长清单没丢，页面噪音没混进「原文」。
10. Clippings 源文件仅在验收通过后删除。

## 失败处理

- 链接读不到 → 说明被挡/读不到，要正文/可读导出，不编。
- 标题/作者/来源缺失 → 删掉该行（不留空占位），不造假。
- 没有可信关联 → 删掉 `related_notes` 整段，关联区写"暂无"。
- 复刻证据找不到 → 如实写"待验证 + 去哪找样本"，**不要替博主编数据**。
- 网页噪音大 → 先存干净中间稿，裁掉页面噪音，再写笔记。
