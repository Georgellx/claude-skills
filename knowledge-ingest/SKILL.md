# 知识入库官（Knowledge Ingest）

把用户发来的文章 / 链接 / 长文本，**经过价值评估后**消化进本地 Obsidian 知识库：该保留的原文保留，该重写的重写，提示词归提示词库，分类不够就以架构师视角补上，最后留下可检索、可双链、可复用的资产。

这个 skill 的灵魂是两件事：**(1) 先判断价值再决定怎么存**；**(2) 一切产物各归其位，单一来源、不重复。**

## 铁律

1. **先分诊，后入库。** 任何文章先给一个"价值档位 + 一句理由"，再决定保留全文 / 重写 / 只存提炼。让用户看到判断，不黑箱。
2. **各归其位，单一来源。** 文章正文 → `文章笔记`；可复制的提示词 → `提示词目录`。提示词**只存一份**在提示词库，文章里只放指针双链（`[[01-内容写作]]`），绝不两边各抄一份。
3. **原文是一手资料，默认保留。** 重写 / 替换要保守，宁可多留。低价值才不存全文，但**永远保留来源链接 + 关键引用**。
4. **不臆造。** 链接抓不到、字段缺失，就如实说、留空、要正文，绝不编。
5. **保守双链。** 只按概念/观点连，不因同作者硬连，0–3 条，低置信就留空。
6. **分类我来扛。** 用户已授权：以"知识体系架构师"视角决定要不要新增分类、归到哪，并在报告里写清理由（方便用户事后否决）。不把这个决定反复丢回给用户。
7. **入库即清理。** 源文件来自 `Clippings` 的，**验收通过后**删除；其它目录的文件不碰。

## 流程

### 第 0 步：去重

先扫一眼 `文章笔记\` 里有没有同主题 / 同源 / 同标题的笔记。已存在就告诉用户、问是更新还是跳过，不要默默建重复笔记。命名可能撞车时，用后缀消歧（如 `内容创作提示词方法论（luke）.md`）。

### 第 1 步：取材

- 粘贴正文 → 直接用。
- 链接 → 抓取正文；分离正文与页面噪音（导航/计数/侧栏/评论）；**先把抓到的原始正文存成中间稿再写笔记**，确认提示词块、代码块、长清单没丢。
- Clippings 文件路径 → 读该文件。
- 抓不到 → 明说"来源读不到"，要用户补正文，**不猜**。

### 第 2 步：价值分诊（核心）

给原文定档（标准见下方「价值分诊标准」），并写一句"价值判断 + 理由"：

- **A 高价值** → 完整保留原文，加「方法解读与验证」层。
- **B 中价值** → 保留原文，另在解读区给一版"干净重写版/要点重述"。
- **C 低价值** → **不存全文**，只存：摘要 + 核心观点 + 关键引用 + 来源链接 + 我的提炼。

### 第 3 步：拆解产物

固定产出：`文章摘要` / `核心观点` / `可复用句子/表达`。
当文章包含方法论、技巧、或可被验证的论断时，**加一节「方法解读与验证」**：大白话重述 + 真伪/适用性点评（哪些有理论支撑、哪些是个人经验、哪些是营销话术）。点评要与内容体量成比例，别为凑而凑。

### 第 4 步：分类路由（架构师视角）

- **文章** → 写进 `文章笔记\`，靠 tag 归类（文章笔记是扁平目录）。
- **提示词** → 文中每条**可直接复制**的提示词，按「提示词目录分类表」路由到对应 `0X-*.md`，用下方「提示词收录格式」追加（**追加，不覆盖**），并在文章的 `05/原文` 处留指针双链。
- **新增分类** → 见「新增分类决策」。命中条件才建，建了就更新 `00-总目录.md` 并在报告里说明。

### 第 5 步：写入与双链

- 文章笔记按下方模板写入 `文章笔记\<标题>.md`。
- `关联笔记` 只连概念/观点强相关的，逐条写明连接理由。
- 提示词追加到对应分类文件末尾。

### 第 6 步：验收 + 清理

- 跑「验收清单」。
- 给用户一份简报：建了/改了哪些文件（可点击链接）、价值档位、分类与新增分类的理由、双链了哪些。
- 源文件在 `Clippings` 的，验收通过后删除（仅此目录）。

```markdown

source_url:

type: 文章笔记

related_notes: [[已有笔记A], [已有笔记B]]   # 0-3 条；不确定就留 []

<紧凑的中文摘要>

# 核心观点

- <值得复用的句子/框架/措辞>

<大白话重述 + 真伪/适用性点评；B/C 档在这里放干净重写版或提炼>

- [[已有笔记A]]：<为什么连>

<A/B 档：完整保留原文（含必要的图片 markdown）；提示词部分用指针双链代替全文复制。

```

## 价值分诊标准

- **A 高价值**：原创观点 / 信息密度高 / 不可替代（一手实践、独家数据、成体系的方法论）。
- **B 中价值**：有干货但啰嗦、AI 味重、软文夹带、结构松散——值得留，但需要我给一版干净重述。
- **C 低价值**：水文 / 纯营销 / 蹭热点 / 一段话能概括 / 全是已知常识。不灌全文，只留提炼 + 链接。

判断时看：有没有"只有这篇才有"的东西？去掉包装后还剩多少干货？读者拿走后能不能复用？

## 新增分类决策（架构师视角，用户已授权我来定）

默认**不新建**——优先塞进现有分类，分类膨胀是知识库头号杀手。

只有同时满足才新建：
1. 内容确实塞不进任何现有分类（不是"沾边"，是"错位"）；
2. 大概率会**重复出现**（不是孤例）；
3. 新分类边界清晰、和现有分类不重叠。

新建对象：
- 新的**提示词用途** → 在 `提示词目录\` 建 `08-xxx.md`（沿用现有文件结构），并在 `00-总目录.md` 的表格 + 快速入口登记。
- 新的**知识领域**（文章层面）→ 一般只加 tag 即可；只有当出现一整个新领域、且会持续积累时，才在 `知识资产\` 下开新顶层板块。

新建后必须在简报里写明："新增了 X 分类，因为……"，让用户能一眼否决。

## 编码与写入注意（Windows）

- 写中文务必 UTF-8，写完抽查几个中文词没变成 `?` 或乱码。
- 追加到现有分类文件时**用 Edit 锚定文件末尾追加**，不要 Write 整文件覆盖（会丢已有内容）。
- 路径用反斜杠（Windows），但 markdown 双链里仍用笔记名 `[[标题]]`，不带路径。

## 验收清单

1. 价值档位已给出且有理由。
2. 文章笔记符合固定模板，元数据尽量填全。
3. A/B 档原文已保留；C 档至少有来源链接 + 关键引用。
4. 提示词已路由到正确分类、单一来源、文章里是指针不是副本。
5. tag 3-5 个、利于检索；双链概念可信、不过连。
6. 新增分类（若有）已更新 `00-总目录` 并写明理由。
7. 中文无乱码。
8. 网页导入：原始正文先存中间稿，提示词/代码/长清单没丢，页面噪音没混进 `原文`。
9. Clippings 源文件仅在验收通过后删除。

---

## 失败处理

- 链接读不到 → 说明被挡/读不到，要正文/可读导出，不编。
- 标题/作者/来源缺失 → 留空，不造假。
- 没有可信关联 → `related_notes: []`，关联区写"暂无"。
- 网页噪音大 → 先存干净中间稿，裁掉页面噪音，再写笔记。
